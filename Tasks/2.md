Задача 2: Змінити існуючий список задач

**Опис:**

Дозволити зміну властивості name для існуючого списку.
Змінювати список задач може тільки власник списку або користувач, що має зв’язок зі списком задач.
Включити валідацію, Result-паттерн, логування, DTO та unit-тести.

**Acceptance:**
1. PATCH /api/tasklists/{id} змінює існуючий список і повертає 200 з TaskListDto.
2. Якщо name некоректний, то повертається 400 Bad Request з ErrorStatus.
3. Якщо поточний користувач не має прав, то повертається 403 Forbidden з ErrorStatus.
4. Якщо список не знайдено, то повертається 404 Not Found з ErrorStatus.
5. У БД name оновлено.

**Request:**
```http
PATCH /api/tasklists/{id}

{
  "name": "Updated List"
}
```

**Response (200 OK):**
```json
{
  "id": "673599e8a1b2c3d4e5f6g7h8",
  "name": "Updated List",
  "ownerId": "user123",
  "sharedWith": ["user456"],
  "createdAt": "2025-11-19T10:30:00Z"
}
```

**Що потрібно реалізувати:**
1. Створити DTO для запиту (UpdateTaskListRequest)
2. Додати валідацію користувача
4. Реалізувати метод в TaskListService
5. Реалізувати метод в TaskListRepository
6. Додати endpoint в TaskListsController

**Логіка виконання:**
1. Знайти список задач по `id`
2. **ВАЖЛИВО:** Перевірити права доступу:
   - Користувач має бути власником (`OwnerId == userId`) 
   - АБО мати зв'язок (`SharedWith.Contains(userId)`)
3. Оновити поле `Name`
4. Зберегти зміни в MongoDB
5. Повернути оновлені дані

**Тести:**
1. Сервіс повертає Failure коли name некоректний або користувач не має прав

**Логування:**
1. Warning при спробі оновлення без відповідних прав, Іnfo у випадку успіху

**Swagger:**
1. Додати коментарі для ендпоінта
