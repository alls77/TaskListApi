Задача 1: Створити новий список задач

**Опис:**

Реалізувати флоу створення нового списку задач: API → service → repository → БД.
Включити валідацію, Result-паттерн, логування, DTO та unit-тести.

**Acceptance:**
1. POST /api/tasklists створює новий список і повертає 201 Created з TaskListDto.
2. Якщо name порожній або >255, то повертається 400 Bad Request з ErrorStatus.
3. Створений документ зберігає ownerId, createdAt (UTC).
4. У БД з’являється документ з валідним _id (ObjectId).

**Request:**
```http
POST /api/tasklists

{
  "name": "List"
}
```

**Response (201 Created):**
```json
{
  "id": "673599e8a1b2c3d4e5f6g7h8",
  "name": "List",
  "ownerId": "user123",
  "sharedWith": [],
  "createdAt": "2025-11-19T10:30:00Z"
}
```

**Що потрібно реалізувати:**
1. Створити DTO для запиту (CreateTaskListRequest)
2. Створити DTO для відповіді (TaskListDetailDto)
3. Додати валідацію:
   - Name обов'язковий
   - Name від 1 до 255 символів
   - UserId обов'язковий
4. Реалізувати метод в TaskListService
5. Реалізувати метод в TaskListRepository
6. Додати endpoint в TaskListsController
7. Встановити OwnerId = userId з запиту
8. Встановити CreatedAt = DateTime.UtcNow

**Тести:**
1. Успішне створення списку
2. Валідація Name (порожній, null, > 255 символів)
3. Валідація UserId

**Логування:**
1. Інформаційний лог про створення, помилки при вставці

**Swagger:**
1. Додати коментарі для ендпоінта
