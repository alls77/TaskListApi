Задача 3: Видалити існуючий список задач

**Опис:**

Реалізувати видалення списку задач. Видаляти може лише власник.
Включити валідацію, Result-паттерн, логування, DTO та unit-тести.

**Acceptance:**
1. DELETE /api/tasklists/{id} видаляє поточний список і повертає 204 No Content.
2. Якщо поточний коричтувач не є власником списку (тобто не має прав на видалення), то повертається 403 Forbidden з ErrorStatus.
3. Якщо список не знайдено, то повертається 404 Not Found з ErrorStatus.
4. У логах створено запис про видалення (audit-friendly).

**Request:**
```http
DELETE /api/tasklists/{id}
```

**Response:** `204 No Content` (порожнє тіло)

**Що потрібно реалізувати:**
1. Додати валідацію користувача
2. Реалізувати метод в TaskListService
3. Реалізувати метод в TaskListRepository
4. Додати endpoint в TaskListsController

**Логіка виконання:**
1. Знайти список задач по `id`
2. **ВАЖЛИВО:** Перевірити права доступу:
   - **ТІЛЬКИ** власник може видалити (`OwnerId == userId`)
   - Користувачі зі зв'язком НЕ можуть видаляти
3. Видалити з MongoDB
4. Повернути 204 No Content

**Тести:**
1. Сервіс повертає Failure коли користувач не має прав для видалення
2. Сервіс видаляє список і повертає Success коли користувач має відповідні права

**Логування:**
1. Warning при спробі видалення без відповідних прав, Іnfo у випадку успіху

**Swagger:**
1. Додати коментарі для ендпоінта
