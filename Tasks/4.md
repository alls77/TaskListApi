Задача 4: Отримати один існуючий список задач

**Опис:**

Реалізувати отримання одного списку.
Отримати список задач може тільки власник списку або користувач, що має зв’язок зі списком задач.
Включити валідацію, Result-паттерн, логування, DTO та unit-тести.

**Acceptance:**
1. GET /api/tasklists/{id} отримує список і повертає 200 з TaskListDto.
2. Якщо поточний користувач не має прав, то повертається 403 Forbidden з ErrorStatus.
3. Якщо список не знайдено, то повертається 404 Not Found з ErrorStatus.

**Request:**
```http
GET /api/tasklists/{id}
```

**Response (200 OK):**
```json
{
  "id": "673599e8a1b2c3d4e5f6g7h8",
  "name": "List",
  "ownerId": "user123",
  "sharedWith": ["user456", "user789"],
  "createdAt": "2025-11-19T10:30:00Z"
}
```

**Що потрібно реалізувати:**
1. Реалізувати метод в TaskListService
2. Реалізувати метод в TaskListRepository (або перевикористати існуючий, якщо вже є з попередніх задач)
3. Додати endpoint в TaskListsController

**Логіка виконання:**
1. Знайти список задач по `id`
2. **ВАЖЛИВО:** Перевірити права доступу:
   - Користувач має бути власником (`OwnerId == userId`) 
   - АБО мати зв'язок (`SharedWith.Contains(userId)`)
3. Повернути інформацію про список
   
**Тести:**
1. Сервіс повертає Failure коли користувач не має прав

**Логування:**
1. Warning при спробі отримання без відповідних прав

**Swagger:**
1. Додати коментарі для ендпоінта
